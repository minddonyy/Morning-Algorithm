## 진행방식

```
git clone {현재 레포지터리 주소}
git pull origin main
git checkout -b {브랜치명 (이름)}


git add .
git commit -m "커밋내용은 밑에 안내에 따른다"
git push origin {자기이름 브랜치}
```


## commit message
feat: 새로운 기능 추가
fix: 버그 수정
docs: 문서
style: 포맷팅, 누락된 세미콜론 등
refactor: 코드 리팩토링
test: 테스트 관련
chore: 기타 수정
build: 빌드 시스템 또는 외부 의존성에 영향을 주는 변경
remove: 파일을 삭제
